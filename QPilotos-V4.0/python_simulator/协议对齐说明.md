#### 一致的部分：
1. **通信协议基础**：ZeroMQ+Json消息体，请求回应模型
2. **所有消息类型**：

   - 中性原子：中性原子平台专用格式
   
2. **指令定义**：每个平台描述自己的量子门
   - 超导：RPhi, ECHO, IDLE, CZ等
   - 光量子：X, Y, Z, H, RZ, CNOT等

## Bug修复记录

### Bug #1：build_get_chip_config_ack参数不匹配

**错误信息：**
```
Error handling message: PhotonicAdapter.build_get_chip_config_ack() missing 1 required positional argument: 'chip_config'
```

**问题原因：**
在 `zmq_router_server.py` 中，对于非超导平台（光量子、离子阱、中性原子），调用 `build_get_chip_config_ack` 时缺少 `point_label_list` 参数：

```python
# 修改前（错误）
reply = self.adapter.build_get_chip_config_ack(
    sn, chip_id, chip_config  # 缺少 point_label_list
)
```

但是三个 adapter 的方法签名已经对齐为：
```python
def build_get_chip_config_ack(sn: int, backend: int, point_label_list: list,
                           chip_config: Dict, err_code: int = 0,
                           err_info: str = "") -> str:
```

**修复方案：**
修改 `zmq_router_server.py` 的 `_handle_get_chip_config` 方法，让所有平台都使用相同的调用方式：

```python
# 修改后（正确）
reply = self.adapter.build_get_chip_config_ack(
    sn, chip_id, point_label_list, chip_config
)
```

**修改文件：**
- `PilotPy/python_simulator/zmq_router_server.py`

**影响范围：**
- 光量子平台 (PQPUmesh8)
- 离子阱平台 (IonTrap)
- 中性原子平台 (HanYuan_01)

**问题原因：**

```python

if self.system_type == QuantumSystemType.SUPERCONDUCTING:
    reply = self.adapter.build_heartbeat_ack(
        sn, self.default_chip_id, "simulator_topic"  # 3参数 ✅
    )
elif self.system_type == QuantumSystemType.PHOTONIC:
    reply = self.adapter.build_heartbeat_ack(
        sn, self.default_chip_id  # 2参数 ❌
    )
else:  # Ion Trap and Neutral Atom
    reply = self.adapter.build_heartbeat_ack(
        sn, self.default_chip_id  # 2参数 ❌
    )
```

但是所有 adapter 的方法签名都是：
```python
def build_heartbeat_ack(sn: int, backend: int, topic: str) -> str:
```

**修复方案：**
修改 `zmq_router_server.py` 的 `_handle_heartbeat` 方法，所有平台统一传入3个参数：

```python
# 修改后（正确）
reply = self.adapter.build_heartbeat_ack(
    sn, self.default_chip_id, "simulator_topic"  # 3参数 ✅
)
```

**修改文件：**
- `PilotPy/python_simulator/zmq_router_server.py`
