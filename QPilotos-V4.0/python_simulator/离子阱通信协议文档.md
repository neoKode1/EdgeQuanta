# 司南测控系统通信协议

## 一.通信协议

ZeroMQ+Json消息体。

ZeroMQ是一个成熟高效的第三方通信库，可方便的实现C++和Python程序之间的高效通信。

ZeroMQ有多种工作模式，目前司南和测控系统对接类似远端调用程序，这里使用"请求回应模型"。

## 二.消息定义（Router-Dealer模式）

### 2.1 下发计算任务

**功能：**该消息用于司南系统向测控系统下发计算任务。

**参数定义：**

|**参数描述**|**参数命名**|**类型**|**备注**|**是否为空**|
|:----|:----|:----|:----|:----|
| 消息类型 | MsgType           | String   | MsgTask                             | 不可空   |
| 息序列ID | SN                | Uint32_t | 消息序列号                          | 不可空   |
| 任务ID   | TaskId            | String   |                                     | 不可空   |
| 量子程序 | ConvertQProg      | String   | 测控要执行的指令（离子阱平台专用格式）                    | 不可空   |
|任务优先级|Configure.TaskPriority|Uint32_t|目前只支持2级优先级：<br>0：普通任务<br>1：高优先级|可空 ，默认为0  |
| 重复次数 |Configure.Shot    | Uint32_t | 取值区间：100~10000，超出后默认1000 | 不可空   |
|是否实验模式|Configure.IsExperiment|Bool|true：实验模式<br>false:非实验模式，即普通用户任务模式|可空，该字段为空时，代表普通用户任务模式|
|执行时序|Configure.ClockCycle|Uint32_t|最大线路执行周期，单位：微秒|可空，司南根据测控返回的芯片最大执行周期自动决定是否传递该参数。比如测控返回当前芯片的普通执行周期是100us，最大执行周期是500us，如果一个任务的执行时序超过100us，小于500us，则根据具体线路时序情况，调整该参数，如果用户线路执行时序小于100us，则不需要传递该参数。|
|应用工厂标签|Configure.PointLabel|Uint32_t|暂定为128|不可空|

**返回信息：**

|**参数描述**|**参数命名**|**类型**|**备注**|**是否为空**|
|:----|:----|:----|:----|:----|
| 消息类型   | MsgType  | String   | MsgTaskAck | 不可空                               |
| 消息序列ID | SN       | Uint32_t | 消息序列号 | 不可空                               |
| 错误码     | ErrCode  | Uint32_t | 错误码     | 不可空                               |
| 错误信息   | ErrInfo  | String   | 错误描述   | 无错误时，可为空；发生错误时，不可空 |

**请求示例**

> {
>     "MsgType":"MsgTask",
>     "SN":133,
> 	"TaskId":"11D919FA044846F3B4DF453A827AE901",
> 	"ConvertQProg":"离子阱平台专用指令格式",
> 	"Configure":{
> 		"Shot":1000,
>   "TaskPriority":1,
>   "IsExperiment": true,
>"PointLabel":128
> 	}
> }
**返回示例**

> {
>     "MsgType":"MsgTaskAck",
>     "SN":133,
> 	"ErrCode":2,
> 	"ErrInfo":"configure error."
> }
### 2.2 任务状态查询

功能：司南向测控系统发送任务状态查看信息，用于确定当前任务计算状态。当司南长时间接收不到计算结果时，可通过此消息，向测控查询任务状态，防止异常情况下，司南死等。

参数定义：

|**参数描述**|**参数命名**|**类型**|**备注**|**是否为空**|
|:----|:----|:----|:----|:----|
| 消息类型   | MsgType   | String   |TaskStatus| 不可空   |
| 消息序列ID | SN        | Uint32_t | 消息序列号      | 不可空   |
| 任务ID   | TaskId            | String   |                  | 不可空   |


返回：

|**参数描述**|**参数命名**|**类型**|**备注**|**是否为空**|
|:----|:----|:----|:----|:----|
| 消息类型   | MsgType   | String   |TaskStatusAck| 不可空   |
| 消息序列ID | SN        | Uint32_t |心跳消息序列号| 不可空   |
| 任务ID   | TaskId            | String   |    | 不可空   |
| 任务状态| TaskStatus| Uint32_t|任务状态，参考任务状态定义| 不可空|


**请求示例**

>     "MsgType":"TaskStatus",
>     "SN":10086,
>     "TaskId":"11D919FA044846F3B4DF453A827AE901"
> }
**返回示例**

> {
>     "MsgType":"TaskStatusAck",

> 	 "TaskId":"11D919FA044846F3B4DF453A827AE901",

### 2.6 心跳消息

功能：用户探测司南系统和测控系统的连接状态。由测控系统维护系统服务，司南系统主动发送心跳包，测控收到心跳包后，发送心跳回应。

参数定义：

|**参数描述**|**参数命名**|**类型**|**备注**|**是否为空**|
|:----|:----|:----|:----|:----|
| 消息类型   | MsgType   | String   |MsgHeartbeat | 不可空   |
| 消息序列ID | SN        | Uint32_t | 心跳消息序列号       | 不可空   |
| 时间戳     | TimeStamp | Uint64_t | 64位整形时间戳，毫秒 | 不可空   |
| 芯片ID| ChipID| Uint32_t|芯片名称，例如：72| 不可空|


返回：

|**参数描述**|**参数命名**|**类型**|**备注**|**是否为空**|
|:----|:----|:----|:----|:----|
| 消息类型   | MsgType   | String   |MsgHeartbeatAck      | 不可空   |
| 消息序列ID | SN        | Uint32_t | 心跳消息序列号       | 不可空   |
|后端ID|backend| Uint32_t|后端芯片ID| 不可空|
| 时间戳     | TimeStamp | Uint64_t | 64位整形时间戳，毫秒 | 不可空   |
|订阅主题|Topic|String|订阅消息主题|不可空|


**请求示例**

```json
{
    "MsgType":"MsgHeartbeat",
    "SN":133,
    "ChipID":72,
    "TimeStamp":1638769359507
}
```

**返回示例**

```json
{
    "MsgType":"MsgHeartbeatAck",
    "SN":133,
    "backend":72,
    "TimeStamp":1638769359517,
    "Topic":"Y4-231011-设计验证-72bit_300pin_V9.2.3_Base-3#_|_Y4-231011-设计验证-72bit_300pin_V9.2.3_Base-3#"

```

| 消息序列ID | SN        | Uint32_t |心跳消息序列号| 不可空   |
| 任务ID   | TaskId            | String   |    | 不可空   |
| 任务状态| TaskStatus| Uint32_t|任务状态，参考任务状态定义| 不可空|

示例

```json
{
    "MsgType":"TaskStatus",
```

### 3.4 自动校准开始信息

功能：在测控一体机进行自动校准时，会推送校准比特信息至司南，此时间段内校准中比特不可用

**operation**：为固定字节 calibration_start；

**data**格式如下：

推送消息格式：

|**参数描述**|**参数命名**|**类型**|**备注**|**是否为空**|
|:----|:----|:----|:----|:----|
| Qubit索引 | qubits| String数组|校准后的单比特信息| 不可空|
| 耦合器 | couplers| String数组|耦合器| 不可空|
| Qubit对|pairs| String数组|校准后的比特对信息| 不可空|
| 鉴别器|discriminators| String数组|鉴别器| 不可空|
|工作标签|point_label|Int|用于表示当前校准作用于哪个标签|不可空|

示例

```json
{
    "config_flag": false,
    "qubits":["q0", "q1", "q2", "q3"],
    "couplers":["c0-1", "c11-12", "c2-3"],
    "pairs": ["q0q1", "q2q3"],
    "discriminators":["q0_01.bin", "q1_01.bin", "q2_01.bin", "q3_01.bin"],
    "point_label": 2
}
```


### 3.5 自动校准结束信息

功能：完成校准后会推送完成消息至司南，比特恢复可用。

| 耦合器 | couplers| String数组|耦合器| 不可空|
| Qubit对|pairs| String数组|校准后的比特对信息| 不可空|
| 鉴别器|discriminators| String数组|鉴别器| 不可空|

示例

```json
{
    "qubits":["q0", "q1", "q2", "q3"],
```

### 3.6 芯片维护信息

功能：接收芯片维护开始和结束信息，实现司南自动暂停。**如果维护时间超过2个小时，则司南向云平台推送维护下线通知；如果维护时间不超过2小时，则只有司南暂停，云平台正常上线**。

**operation**：为固定字节 chip_protect；

推送消息格式：

|**参数描述**|**参数命名**|**类型**|**备注**|**是否为空**|
|:----|:----|:----|:----|:----|
| 维护标志| ProtectFlag|bool|true 为维护开始，false维护结束| 不可空|
|维护持续时间| DurativeTime| uint64_t|单位：分钟| 不可空|
|时间戳|LastTime| uint64_t|时间戳| 不可空|

示例

```json
{
  "ProtectFlag":true,
  "DurativeTime":10,
  "LastTime":1705307288685
}
```

## 三.类型定义

### 3.1 任务状态类型

|**枚举**|**枚举值**|**类型说明**|
|:----|:----|:----|:----|
| UNKNOW_STATE| 0| 未知状态   |
| PENDING| 1      | 任务还在排队    |
| RUNNING| 2      | 任务正在运行             |
| NOTASK| 3      | 找不到对应的任务信息 |
| FAILED| 4      | 任务计算失败      |
| SUCCESSED| 5      | 任务计算完成//      |
| RETRY| 6      | 任务重新发送      |
|COMPILING| 7| 任务正在编译|
|COMPILED| 8| 任务编译完成|


### 3.2 错误码类型

|**枚举**|**枚举值**|**类型说明**|
|:----|:----|:----|:----|
| NO_ERROR|0|  无错误 |
| UNDEFINED_ERROR|1|  未知错误|
| TASK_PARAM_ERROR|2|  计算任务参数错误|
| JSON_ERROR|3|  Json错误|
| QUEUE_FULL|4|  任务队列满（司南发送任务超过200条时返回此错误）|

## 四. 指令定义

### 4.1 包含的基础门类型与参数说明

离子阱平台的基础门类型与超导平台不同，ConvertQProg字段保持离子阱平台专用格式，不强制要求与超导一致。

支持的离子阱门包括：MS, Rabi等离子阱特有的量子门操作。
